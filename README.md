# レシピサイト README

## 1. アプリ概要

このアプリは、ユーザー登録・ログイン後にレシピを検索・閲覧するためのWebアプリケーションです。

現在の実装では、ユーザー情報、カテゴリ情報、レシピ情報をPostgreSQLで管理し、レシピ一覧画面でキーワード検索やカテゴリ検索を行う構成になっています。

ただし、現状コードにはコンパイルエラーになる箇所、画面遷移の不整合、URL生成ミス、ログイン後の一覧表示不足などがあるため、まずは既存機能を正常に動作させる修正が必要です。

## 7. データベース設計

### users テーブル

ユーザー情報を管理するテーブルです。

```sql
CREATE TABLE users
(
   id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
   name VARCHAR(20),
   email VARCHAR(255),
   password VARCHAR(50)
);
```

| カラム名 | 型 | 制約 | 内容 |
|---|---|---|---|
| id | INTEGER | PRIMARY KEY / 自動採番 | ユーザーID |
| name | VARCHAR(20) | なし | ユーザー名 |
| email | VARCHAR(255) | なし | メールアドレス |
| password | VARCHAR(50) | なし | パスワード |

---

### categories テーブル

レシピカテゴリを管理するテーブルです。

```sql
CREATE TABLE categories
(
   id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
   name VARCHAR(20)
);
```

| カラム名 | 型 | 制約 | 内容 |
|---|---|---|
| id | INTEGER | PRIMARY KEY / 自動採番 | カテゴリID |
| name | VARCHAR(20) | なし | カテゴリ名 |

---

### recipes テーブル

レシピ情報を管理するテーブルです。

```sql
CREATE TABLE recipes
(
   id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
   user_id INTEGER REFERENCES users(id),
   category_id INTEGER REFERENCES categories(id),
   name VARCHAR(50),
   recipe TEXT
);
```

| カラム名 | 型 | 制約 | 内容 |
|---|---|---|---|
| id | INTEGER | PRIMARY KEY / 自動採番 | レシピID |
| user_id | INTEGER | FOREIGN KEY | 登録ユーザーID |
| category_id | INTEGER | FOREIGN KEY | カテゴリID |
| name | VARCHAR(50) | なし | レシピ名 |
| recipe | TEXT | なし | レシピ本文 |

## 11. 処理の流れ

### 11-1. ログイン処理

```text
ログイン画面を表示
  ↓
メールアドレスとパスワードを入力
  ↓
UserRepositoryでユーザー検索
  ↓
一致するユーザーが存在するか確認
  ↓
存在する場合、Accountにメールアドレスを保存
  ↓
現在はRecipes.htmlを直接返す
```

本来は、ログイン成功後に以下のようにリダイレクトする方が安全です。

```text
redirect:/recipes
```

---

### 11-2. 新規会員登録処理

```text
新規会員登録画面を表示
  ↓
名前、メールアドレス、パスワードを入力
  ↓
入力チェック
  ↓
メールアドレス重複チェック
  ↓
問題がなければusersテーブルに保存
  ↓
ログイン画面へ戻る
```

---

### 11-3. レシピ一覧・検索処理

```text
/recipes へアクセス
  ↓
CategoryRepositoryでカテゴリ一覧を取得
  ↓
categoryId がある場合はカテゴリ検索
  ↓
keyword がある場合はキーワード検索
  ↓
どちらもない場合は全件表示
  ↓
Recipes.htmlに recipes / categories / keyword / categoryId を渡す
```

## 12. application.properties

現在のDB接続設定は以下です。

```properties
spring.datasource.driver-class-name=org.postgresql.Driver
spring.datasource.url=jdbc:postgresql:personal_dev_11686_takahashi_io
spring.datasource.username=student
spring.datasource.password=himitu
spring.jpa.show-sql=true
spring.sql.init.mode=never
```

### 注意点

`spring.sql.init.mode=never` のため、`schema.sql` と `data.sql` は自動実行されません。

初回セットアップ時は、PostgreSQLに手動でSQLを実行するか、開発初期のみ以下に変更して実行します。

```properties
spring.sql.init.mode=always
```

ただし、`always` にすると起動時にSQLが再実行されるため、既存データを消したくない場合は注意してください。

# 14. 現在の問題点

## 14-1. コンパイルエラーになる可能性が高い箇所

`RecipeController.java` に以下の未完成メソッドがあります。

```java
@GetMapping("/recipes/detail/{id}")
public String detail() {
}
```

`String` を返すメソッドなのに `return` がありません。

この状態では、Javaのコンパイルエラーになります。

### 対応方針

最低限、以下のように戻り値を返す必要があります。

```java
@GetMapping("/recipes/detail/{id}")
public String detail() {
    return "recipeDetail";
}
```

ただし、詳細画面を作るなら、`id` を受け取って対象レシピを検索し、`recipeDetail.html` に渡す必要があります。

---

## 14-2. ログイン後に `Recipes.html` を直接返している

`UserController.java` のログイン成功時が以下になっています。

```java
return "Recipes";
```

この場合、`RecipeController` の `/recipes` 処理を通らないため、`categories` や `recipes` がModelに入らない可能性があります。

### 対応方針

ログイン成功時は以下に変更する方が安全です。

```java
return "redirect:/recipes";
```

---

## 14-3. ログイン情報としてAccountにemailしか保存していない

現在のログイン成功時は以下です。

```java
account.setEmail(user.getEmail());
```

しかし、レシピ登録やユーザーごとの絞り込みを行う場合は、ユーザーIDと名前も必要になります。

### 対応方針

`Account.java` に `id` を追加し、ログイン成功時に以下を保存します。

```java
account.setId(user.getId());
account.setName(user.getName());
account.setEmail(user.getEmail());
```

---

## 14-4. 未ログインでも `/recipes` にアクセスできる

現在、`RecipeController.java` 側でログイン確認をしていないため、未ログイン状態でも `/recipes` を開ける可能性があります。

### 対応方針

`RecipeController` に `Account` をDIし、各処理の最初でログイン確認を行います。

```java
if (account.getId() == null) {
    return "redirect:/login";
}
```

---

## 14-5. カテゴリ検索リンクのURLが誤っている

現在の `Recipes.html` には以下のような記述があります。

```html
<a th:href="'/recipes'+${category.name}" th:text="${category.name}"></a>
```

この場合、生成されるURLは以下のようになります。

```text
/recipes丼もの
```

これはControllerの `/recipes` に対応していません。

### 対応方針

カテゴリIDをクエリパラメータとして渡す必要があります。

```html
<a th:href="@{/recipes(categoryId=${category.id})}" th:text="${category.name}"></a>
```

---

## 14-6. レシピ詳細リンクのURLが誤っている

現在の `Recipes.html` には以下のような記述があります。


## 14-8. Recipe.java のgetter/setterに不備がある

現在の `Recipe.java` には以下のようなメソッドがあります。

```java
public String getrecipes() {
    return recipe;
}

public void setId(String recipe) {
    this.recipe = recipe;
}
```

問題点は以下です。

| 現在 | 問題 |
|---|---|
| `getrecipes()` | JavaBeansの命名規則として不自然 |
| `setId(String recipe)` | メソッド名が誤っている。recipeをセットするなら `setRecipe` が正しい |

### 対応方針

以下に修正します。

```java
public String getRecipe() {
    return recipe;
}

public void setRecipe(String recipe) {
    this.recipe = recipe;
}
```

# 19. 最終まとめ

このアプリは、ユーザー登録・ログイン後にレシピを検索・閲覧するためのレシピサイトです。

現在、ユーザー登録、ログイン、レシピ一覧表示、キーワード検索の基本構成は作られています。

一方で、`RecipeController.java` の詳細メソッドに戻り値がないため、まずコンパイルエラーを解消する必要があります。

次に、ログイン後の遷移を `redirect:/recipes` に変更し、カテゴリ一覧とレシピ一覧が正しく表示されるようにします。

その後、カテゴリ検索リンク、レシピ詳細リンク、詳細画面、ログイン制御、AccountへのユーザーID保持を順番に修正することで、マスト機能に近づけることができます。
